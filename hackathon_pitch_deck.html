PROBLEM DEFINITION

문제 이해

기업 원문 문제

CS팀이 이메일·채널톡·미팅으로 산발적으로 들어오는 VoC를 수작업으로 엑셀에 정리하고 있어, 유형별 패턴 파악이 어렵고 비즈니스 의사결정에 활용하지 못하고 있다.

내가 재정의한 문제

"이번 주 리포트 1장 잘 만들기"가 진짜 문제가 아니라, CS팀이 매주 반복하는 수작업과 신규 담당자가 규제 맥락을 몰라 문의조차 이해하기 어려운 것이 진짜 페인포인트. 그래서 "새 CSV만 넣으면 매주 같은 품질로 나오는 파이프라인"으로 문제를 좁혔다.

문제의 크기 (Before)

CS 담당자 하루 평균 20~30건 문의를 수작업 분류, 신규 담당자는 규제 맥락 파악에만 시간 소요, 제품팀 전달용 인사이트 정리에 주 1~2시간 소요 (문제 원문 가상 시나리오 기준).

스코프 — 푼 것 / 안 푼 것

O 4유형 분류·집계·인사이트(Basic), 재사용 파이프라인·주간리포트(Standard), 제품기획안·우선순위설계·구글시트연동·슬래시커맨드 패키징(Challenge+도전)
X 이메일·채널톡 실시간 연동, 자동응답/CS티켓 처리, 다국어 VoC, 실제 고객데이터 — 문제 범위상 의도적으로 제외

APPROACH & DECISION

해결 접근 & 의사결정

접근 가설 / 핵심 흐름

VoC CSV 업로드 → 날짜·결측·중복 정제 → 규칙 기반 분류(불만→요청→칭찬→문의 우선순위) → 집계 → 인사이트 생성 → 주간 리포트·제품 기획안 자동 작성까지 한 번에 이어지는 흐름으로 설계.

버린 대안 & 이유

채널 결측을 최빈 채널로 채우는 방식은 실제 채널 분포를 왜곡할 위험이 커 배제하고 '미기재'로 명시. LLM API로 바로 분류하는 방식도 검토했지만 "별도 API 키 없이 대화 안에서 완료"라는 제약 때문에 키워드 우선순위 규칙 방식을 채택.

★ 막혔던 지점 & 돌파 방법

일부러 만든 오염 데이터로 스트레스 테스트를 하다가 두 가지를 실제로 발견했다. ① content가 빈 값인 행들이 서로 "중복"으로 오인되어 자동 삭제되는 버그 — 중복 판정에서 빈 값을 제외하도록 수정. ② VoC 원문에 <script> 태그를 넣었더니 리포트 화면에 그대로 삽입되는 저장형 XSS 취약점 — 표·문단·인사이트 카드에 escape를 적용해 막았다. 둘 다 원본 데이터 결과에는 회귀 없음을 재검증했다.

AI vs 내 판단

코드 작성·전체 재실행·오염 데이터 스트레스 테스트는 Claude Code에 맡기고, 분류 우선순위·우선순위 산정식(빈도+임팩트+긴급도+규제연관성-난이도)·경계 케이스 처리 기준은 내가 직접 설계하고 검토·승인했다.

APPENDIX

(별첨) 기술 & 논리 상세

기술 검증을 원하는 담당자만 여기로 — 본문은 비즈니스 언어로만

워크플로우 구조 / 파이프라인

clean.py → classify.py → aggregate.py → insight.py → report.py → product_plan.py, run_pipeline.py가 6단계를 순서대로 실행. app.py는 pipeline.run_analysis()를 그대로 호출해 CLI와 대시보드가 동일 로직을 공유.

사용 도구 & 주요 노드

Claude Code(파이프라인 작성·실행·검증), Streamlit(대시보드), pandas(데이터 처리), gspread(선택적 구글시트 연동), Claude in Chrome(실사용 검증).

데이터 처리 과정

CSV를 문자열 그대로 로드 → 날짜 3종 형식(YYYY-MM-DD/YYYY-MM-DD/M월D일) 정규화 → 결측 플래그+채움 → content 완전 중복만 제거(빈 값은 제외) → 우선순위 규칙(불만>요청>칭찬>문의)으로 분류 → urgency·customer_impact 산정 → 집계 → 회사·산업 컨텍스트를 인용한 인사이트 생성 → 리포트·기획안 렌더링.
