## 5. 의사결정 로그 요약

전체 근거는 `output/decisions.md`(방법론 문서, 6장) 및 `decisions.md`(진행 로그)에 기록. 핵심만 요약:

**분류 판단 우선순위**: 불만 → 기능 요청 → 칭찬 → 일반 문의 순으로 검사. 불만 신호가 있으면 다른 신호가 섞여 있어도 불만이 최우선이며, 상세 맥락은 `sub_intent`에 남긴다 (예: 불만+요청 → `불만(개선요청)`).

**경계 케이스 처리**:
| 혼합 상황 | 처리 결과 |
|-----------|-----------|
| 불만 + 요청 (예: id043 "제한적입니다...요청드립니다") | `불만`, sub_intent=`개선요청` |
| 문의 + 요청 (예: id006 "설명하는 가이드 문서를...") | `기능 요청`, sub_intent=`가이드요청`/`기능요청` |
| 칭찬 + 개선 제안 | 긍정·요청 표현 강도를 비교해 더 강한 쪽으로 분류 |
| id001/031 완전 중복 | 최초 접수 건(id001)만 유지 |

**urgency 판단**: High(긴급/마감/반려/오류/안 됩니다 등) > Medium(비용/인증/검증/차이 등) > Low(정보 확인) 순.
"감사"(칭찬)와 "내부 감사"(규제 감사)의 동형어, "오류가 줄었습니다"(칭찬)처럼 해소 문맥이 오는 경우는 가드 로직으로 오탐을 방지했다.

## [Standard] 파이프라인 사용법

> 새 VoC CSV를 넣으면 동일한 리포트를 자동 생성하는 방법

**사용 방법 1 — 스크립트 직접 실행**
```bash
pip install -r requirements.txt
python src/run_pipeline.py --input data/voc.csv --output output/original --dataset-label original --default-year 2026
```
`--input`에 새 CSV 경로만 바꿔 넣으면 정제→분류→집계→인사이트→주간 리포트→제품 개선 기획안까지
동일한 품질로 자동 생성된다. 입력 파일이 없거나 필수 컬럼이 누락되면 트레이스백 없이 원인을 바로 알려주고
종료하며, 같은 `--output` 폴더를 다른 `--dataset-label`로 재실행하면 원본/증강 결과 혼재를 막기 위해
에러를 낸다.

**사용 방법 2 — Claude Code 슬래시 커맨드**
```
/voc-report data/voc.csv output/original original
```
(`.claude/commands/voc-report.md`에 정의됨 — 파이프라인 실행 후 결과를 자동으로 요약해서 알려준다)

**재현성 검증**: 제공된 원본 데이터 외에, 실무 운영 규모(200건)를 가정한 synthetic 데이터로도 파이프라인을
검증했다. `python src/generate_synthetic_voc.py --base data/voc.csv --output data/voc_augmented.csv --n 200 --seed 42`
로 생성 후 동일 파이프라인을 실행한 결과, voc_type/topic/urgency 분류 정확도 **100%**를 확인했다
(`output/augmented/synthetic_data_note.md`).

## [Challenge] 제품 개선 기획안

우선순위 산식: `priority_score = frequency_score + impact_score + urgency_score + regulation_score - effort_score`

### 기획안 1. CBAM 신고·인증서 제출 어시스턴트 (Total Score: 11)
- **문제 정의**: CBAM 신고 기한·인증서 구매 절차·EU 세관 제출 양식에 대한 문의·불만이 4건 반복 접수. 정보 확인성 문의(id001)부터 EU 세관 반려(id010), 협력사 협조 문제(id023), 마감 초과로 수출 계약 위험(id045)까지 위험도가 점진적으로 심각해지는 패턴.
- **근거 VoC**: 4건 — id010(불만/High), id045(불만/High), id006(기능 요청/Medium), id023(기능 요청/Low)
- **개선 제안**: 신고 기한·인증서 구매·제출 양식 체크리스트를 제공하는 'CBAM 신고 어시스턴트'. D-30/D-7 알림 배너 + 제출 전 필수 항목 자동 검증 + urgency High 건 컨설팅팀 자동 에스컬레이션.
- **우선순위**: Frequency 3(4건) + Impact 3(OEM/1차 협력사) + Urgency 3(High 포함) + Regulation 3(CBAM) − Effort 1(문서/가이드 수준) = **11점**. 임팩트·긴급도·규제연관성이 모두 최고점인데 구현 난이도는 가장 낮아 최우선 과제로 선정.

### 기획안 2. 계산 로직 투명성·검증 강화 (Total Score: 9)
- **문제 정의**: 카본링크 계산 결과가 외부 산정값·경쟁사 플랫폼과 다르다는 지적 3건. 20% 차이(id014), 30% 이상 차이(id033), IPCC 기준 미반영 의심(id019), 인증 제출용 데이터 내보내기 실패(id027)까지 계산 신뢰성 문제가 반복.
- **근거 VoC**: 3건 — id027(불만/High), id014(불만/Medium), id019(불만/Low)
- **개선 제안**: 계산 근거를 항목별로 펼쳐보는 '계산 상세 보기' 화면 + ISO 14064 등 인증기관 제출용 표준 포맷 내보내기 + 차이 임계치(15%) 초과 시 자동 검토 티켓 생성.
- **우선순위**: Frequency 2(3건) + Impact 3(1차/2차 협력사) + Urgency 3(High 포함) + Regulation 3(Scope1/검증인증) − Effort 2(화면/문서화) = **9점**. 계산 신뢰성은 제품 핵심 신뢰 지표이므로 방치 시 이탈 리스크가 크다고 판단.
