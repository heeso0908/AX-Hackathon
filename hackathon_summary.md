# 해커톤 작업 요약 — [01] 리뉴어스랩 VoC 분류·인사이트 리포트

발표 자료 제작용 요약 문서. 2026-07-02 기준.

---

## 1. 과제 개요

- **기업**: 리뉴어스랩 — 공급망 탄소 데이터 자동화 플랫폼 **카본링크(CarbonLink)**. 현대차·BMW·테슬라 등 글로벌 OEM 공급망 협력사 310여 곳 고객. CBAM(EU 탄소국경조정제도)·CSRD(기업 지속가능성 보고 지침) 대응 솔루션 제공. TIPS 선정 스타트업.
- **문제 상황**: VoC가 이메일·채널톡·설문·미팅 등으로 산발적으로 유입되고, CS팀이 수작업 엑셀 정리에 의존 → 유형별 패턴 파악과 비즈니스 의사결정 활용이 어려움. CBAM 확정 시행(2026-01)·CSRD 적용 확대와 맞물려 문의·불만이 급증하는 시점.
- **미션**: `data/voc.csv`(45건)를 분석해 (1) 불만·기능 요청·칭찬·일반 문의 4유형 분류, (2) 유형별 건수 집계, (3) CSRD·CBAM 맥락 비즈니스 인사이트 3개 이상 도출.
- **채점 구조(누적형)**: Basic 100점(분류 40·집계 20·인사이트 30·제출형식 10) → Standard +30점(파이프라인 재현성·주간 리포트 설계) → Challenge +30점(제품 개선 기획안·우선순위 설계). 만점 160점.

## 2. 접근 방식 — "리포트 1건"이 아니라 "재사용 가능한 파이프라인"으로 설계

Basic 요구사항(분류표+인사이트 문서 1개)을 넘어, **새 CSV를 넣어도 정제→분류→집계→인사이트→리포트→기획안이 동일 품질로 자동 생성되는 파이프라인**을 목표로 잡았다. 이유: 실제 CS팀 운영 시나리오에서는 매주 새 VoC가 들어오므로, 1회성 분석보다 재현 가능한 도구가 더 실무적 가치가 크다고 판단.

```
정제 → 분류 → 집계 → 인사이트 → 주간 리포트 → 제품 개선 기획안
(clean)  (classify)  (aggregate)  (insight)  (report)      (product_plan)
```

`src/run_pipeline.py` 한 번 실행으로 8개 산출물이 `output/<label>/`에 생성되며, CLI(`run_pipeline.py`)와 Streamlit 대시보드(`app.py`)가 동일한 `src/pipeline.py`의 `run_analysis()`를 공유 — 로직 이중 구현 없음.

## 4. 분류 로직 — 4유형 + 혼합 케이스 처리

우선순위 **불만 → 기능 요청 → 칭찬 → 일반 문의** 순으로 판단 (불만 신호가 있으면 다른 신호가 섞여도 불만이 최우선, 나머지는 `sub_intent`에 보조 기록).

- 예: id 043 "보고서 커스터마이징 기능이 너무 제한적입니다. 개선 요청드립니다." → 불만(개선요청)으로 분류. 이탈 위험(불만 신호)이 개선 요청보다 우선한다는 원칙 적용.
- 키워드 매칭 기반 rule-based 분류(`src/classify.py`)이며, 개발 중 "설명해주세요"(정보 확인)를 기능 요청으로, "오류가 줄었습니다"(칭찬)를 불만으로 오인식하는 사례를 발견해 가드 로직으로 보정.

**분류 결과 (44건, 중복 제거 후)**

| 유형 | 건수 | 비율 |
|---|---|---|
| 불만 | 12 | 27.3% |
| 기능 요청 | 11 | 25.0% |
| 칭찬 | 8 | 18.2% |
| 일반 문의 | 13 | 29.5% |
| **합계** | **44** | **100%** |

## 5. 탄소규제 맥락 인사이트 (5개 — 요구사항 3개 초과 달성)

1. **CBAM 확정 시행 리스크 구간 진입**: 규제 관련 VoC 19건(43.2%), CBAM 맥락이 10건으로 최다. EU 세관 반려(id010)·마감 초과(id045) 등 계약 리스크 사례 포함 → CBAM 신고 절차 표준 가이드 우선 제작 필요.
2. **Scope 3/공급망 데이터 제품 개선 신호**: 협력사 수백 곳의 Scope 3 데이터 수동 입력 구조가 CS 반복 문의의 근본 원인 → 협력사 데이터 일괄 요청·자동 수집 기능 검토 필요.
3. **단순 정보 확인형 문의의 FAQ 전환 여지**: 15건(34.1%)이 절차 반복 질문 → 지식베이스 전환 시 신규 CS 담당자 온보딩 부담 절감.
4. **OEM/1차 협력사 고긴급도 불만 = 이탈 위험 신호**: urgency High 3건이 모두 계약·업무 리스크 직결 → SLA 24시간 전담 대응 필요.
5. **칭찬 VoC 중 세일즈 레퍼런스 소재**: 정량적 성과가 담긴 후기 3건(업무시간 절반 절감 등) → 별도 인터뷰 없이 마케팅 콘텐츠로 활용 가능.

## 6. Standard/Challenge 확장 구현

- **파이프라인 재현성**: `--input/--output/--dataset-label` 옵션으로 새 CSV 즉시 재실행. 필수 컬럼 누락 등 예외 상황도 트레이스백 없이 원인 안내.
- **주간 리포트 설계**: Executive Summary~다음 액션까지 11개 섹션. High urgency Queue(CS팀)·제품팀 전달 후보(제품팀)·규제 캘린더 연계 인사이트(리더/PM) 등 독자별 섹션 설계.
- **제품 개선 기획안**: `priority_score = frequency + impact + urgency + regulation - effort` 산식을 직접 설계해 6개 클러스터(CBAM 신고, Scope3 자동화, 보고서 커스터마이징, 계산 로직/검증, API 연동, FAQ/온보딩) 채점 후 상위 2건 작성.
- **구글시트 연동**: 기본(항상 실행, 자격증명 불필요) TSV 내보내기 + opt-in 서비스 계정 자동 기입. 실제 구글시트 자동 기입까지 검증 완료.
- **Skill 패키징**: `/voc-report` 슬래시 커맨드로 정제~기획안 전체 파이프라인 재현 가능.
- **Streamlit 대시보드**: CarbonLink 스타일(화이트+그린 포인트) UI, 6개 탭(Overview/Basic/Standard/Challenge/Upload CSV/Decisions), CSV 업로드 시 즉시 분석.

## 7. 검증 — 재현성·강건성 실측

- **재현성 검증**: synthetic 데이터(200건, `voc_augmented.csv`)로 파이프라인 재실행 → 245건 입력, 237건 정제, High urgency 31건, 검증용 `expected_*` 컬럼 기준 분류 정확도 **100%** 확인.
- **강건성(오염 데이터) 검증**: 원본 45건 + 의도적 결함 20건 = 65건 더미 데이터(`voc_dirty_test.csv`)로 스트레스 테스트. 날짜 완전 결측·존재하지 않는 날짜·content 완전 공백·id 중복·정의되지 않은 customer_type·이모지 등 포함 → **크래시 없이 60건 정제 완료**, 품질 플래그 전부 기대값과 일치.
  - 이 과정에서 "content가 빈 값인 행끼리 서로 중복으로 오인해 삭제되는" 실제 버그를 발견해 `src/clean.py`의 중복 판정 로직을 수정 (`clean["content"] != ""` 조건 추가). 원본 데이터에는 영향 없음을 회귀 테스트로 확인.
- **최종 검증 리포트**(`output/verification-report.md`): 로컬 코드를 직접 재실행해 README/제출물 주장과 실제 산출물 일치 확인. 배포된 Streamlit 앱과 로컬 실행 결과 수치까지 브라우저로 직접 대조해 일치 확인. Basic 100 + Standard 30 + Challenge 30 = **160점(만점) 전 항목 충족** 판정.

## 8. 한계와 개선 방향 (발표 시 리스크/향후 계획 슬라이드용)

- Rule-based 분류라 문맥을 완전히 이해하지 못함 — 어미 차이, 키워드 중의성으로 인한 오분류 사례를 개발 중 다수 발견해 가드 로직으로 대응했으나 근본적으로는 LLM API 연동 시 분류 근거 문장 품질을 한 단계 끌어올릴 수 있음.
- 실 운영 시 CS 담당자 피드백을 반영한 키워드 사전 주기적 업데이트 프로세스 필요.
- 구글시트 연동은 opt-in 방식 — Slack/CRM까지 연동하면 리포트 생성~배포 완전 자동화 가능(현재 범위 밖).
- Effort(구현 난이도) 점수는 클러스터 성격에 따른 고정 배정 — 데이터로 자동 산출은 아님.
